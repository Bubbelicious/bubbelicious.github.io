NärCon Panda

This website is a recrutement site for the big event organization NärCon.
I have personally been a volonteer at their events since 2012, and unfortunately the website has not changed much since then, and is very outdated. Also, considdering the current regulations about accessability and availability online, NärCon panda does not meet the requirement that are recommended.
This is a problem since NärCon is very adamant about accepting everyone no matter of disability or other. I have therefore taken upon myself to redesigning the site to meet the WCAG requirement as well as updating it and adapting it with a mobile design (something the current one lacks).
Project still in progress.

Research

Since I have personal experience with the site, not much actual research was needed to collect the data I needed. I have heard many complaints about the site, and started noting them down when I decided that it needed to be changed. I completed these notes with a few unstructural interviews to get the whole picture.
I used the tool Lighthouse, as well as Siteimprove Accessability Checker, to get the exact data of what needs to be adjusted for the site to be up to WCAG requirement.

"I can never remember how to find the profile settings..."

"The private message system is driving me crazy! I always click on the wrong thing."

"Why can't I search for the things I actually need to find?"

Result of Research

From my research I learned that while the graphics are outdated, they hold up to the WCAG requirement for color and contrast but the site is severly underworked in terms of accessability in the code. The site is not adapted for people with sight disabilities or other who need to tab their way through the site. This is not something I can personally do anything about, but I will bring it up none the less.

On the other hand, the personal interviews gave an entirely other perspective. The users are often frustrated with the navigation on the site, and find it hard to locate what they are looking for.
While the navigation seems to be the biggest concern for users, they all have other problems that they mention. Some reaccuring ones are: Private Messages, Profile settings, Applications, Search, and Discussions.

First Design

One things I had to keep in mind when redesigning the site was that NärCon has a very recognizable profile, and I had to make sure that my design incorporated. The website was to be given a desperatly needed update, but it still had to be recognized as one of NärCon's sites.
Tidying up the navigation was my main concern, as well as figuring out how the site would look as it adapted itself to a mobile user. These would be the first things I worked on before going deeper into the functions on the site.

Start page

For the start page, and log in function, I tidied up the navigation to make it easier to find what you are looking for, and moved the functions of creating an account and forgotten password into the log in function since you don't want to have to look for it if you need it.

While I kept the idea of a background image, I made sure to darken it enough for it to not be distracting or annoying for the eyes.

I made sure that the design for the mobile adaption was as close to the desktop site as possible, since it needs to still read as the same website.

I decided to make the navigation into a hamburger menu to make it as easy to navigate as possible without making it cluttered.

Home page

The navigation menu on the home page is very overwhelming and confusing.
In my design I decided to keep the menu from the start page since the functions still are there but for some of them under different names in the current site. My design also includes an easier to locate profile, as well as all the functions that are included in that, and grouping together the application function with the teams that are available to make it easier to find.

The mobile adaption to the home page follows a simliar pattern as the start page, trying to make it as recognizable from the desktop site as possible. The menu is meant to be easy to navigate and not be too cluttered while trying to fit the needed functions in it.

Home Page functions

The sites functions on messages and notifications used to have their own page, which according to users was not at all user friendly, but in my design I decided to give the users the option on viewing and gaining access to their messages directly from the home page. My inspiration for this is fairly obvious (Facebook), but it is a lot more user friendly and you can still go to the page specifically for messages and notifications if you want to view all of them.
